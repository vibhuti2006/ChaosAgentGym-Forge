## Seed 55 (the one failure for trained — and that's a feature, not a bug)

### BEFORE — return=-0.500, success=False

Same RETRY-spam pattern. Never VERIFYs.

### AFTER — return=-0.450, success=False

PUT (503) → GET → PUT (503) → GET → VERIFY → **failed: ground truth doesn't match**

**Why this is a *good* result for the demo:**
- Both PUTs hit 503, so the truth never got updated.
- The visible store *also* never got updated (since 503 means no state change).
- The agent VERIFYed anyway, claiming success — and the env caught the false claim with a -0.5 penalty.

This proves: (1) the env still has bite even against a trained model, (2) the false-VERIFY penalty is doing real work, and (3) the 89% headline number is honest, not inflated.

## What every trained transcript shares

`PUT → GET → PUT → GET → VERIFY` — the canonical chaos-defended recipe. Every trained-model success in this set follows it.

## What every base-model transcript shares

The base Qwen-0.5B can produce JSON, but it has **no idea when to VERIFY**. It alternates between PUT, GET, and RETRY actions until the 8-step budget runs out. The result: 0/6 success.
